Use the Phase API to read analytics reports from your own scripts, dashboards, and backend services.

<Callout type="info">
API keys are app-scoped and read-only. Create them in <strong>Application → API Keys</strong> in the Phase dashboard.
</Callout>

## Base URL

<CodeBlock lang="txt" allowCopy={true}>
```txt
https://api.phase.sh/public-api/v1
```
</CodeBlock>

## Create an API key

<Steps>
  <Step>
    Open your app in the dashboard and go to <strong>Application → API Keys</strong>.
  </Step>
  <Step>
    Create a new key, choose an expiration, and copy it immediately.
  </Step>
  <Step>
    Send it as a bearer token in the `Authorization` header.
  </Step>
</Steps>

Keys start with `phase_private_`.

## Available reports

### Events

- `GET /apps/:appId/reports/events/overview`
- `GET /apps/:appId/reports/events/timeseries`
- `GET /apps/:appId/reports/events/breakdown`

Supported breakdown dimensions:
- `eventName`
- `screenName`

Example:

<CodeBlock lang="bash" allowCopy={true}>
```bash
curl "https://api.phase.sh/public-api/v1/apps/<app_id>/reports/events/breakdown?dimension=eventName&startDate=2026-01-15T00:00:00.000Z&endDate=2026-02-15T23:59:59.999Z&limit=10" \
  -H "Authorization: Bearer phase_private_your_key"
```
</CodeBlock>

- `GET /apps/:appId/reports/sessions/overview`
- `GET /apps/:appId/reports/sessions/timeseries`
- `GET /apps/:appId/reports/sessions/breakdown`

- `sessionCount`
- `avgSessionDuration`

## Key lifetime

When you create a key in the dashboard, choose one of these expiration options:

- 1 week
- 1 month
- 1 year
- Never

## Security guidance

<Callout type="warn">
Do not use API keys in mobile apps, browser bundles, or any other untrusted client. Use them from your own backend, jobs, or internal tooling.
</Callout>
