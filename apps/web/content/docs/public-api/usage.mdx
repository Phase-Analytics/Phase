Use the Phase API to read analytics reports from your own scripts, dashboards, and backend services.

<Callout type="info">
API keys are app-scoped and read-only. Create them in <strong>Application → API Keys</strong> in the Phase dashboard.
</Callout>

## Base URL

<CodeBlock lang="txt" allowCopy={true}>
```txt
https://api.phase.sh/public-api/v1
```
</CodeBlock>

## Quick start

<Steps>
  <Step>
    Open your app in the dashboard and go to <strong>Application → API Keys</strong>.
  </Step>
  <Step>
    Create a new key, choose an expiration, and copy it immediately.
  </Step>
  <Step>
    Call a report endpoint with your app ID and the key in the `Authorization` header.
  </Step>
</Steps>

Keys start with `phase_private_`.

## First request

<Tabs defaultValue="curl" groupId="api-first-request" persist>
  <TabsList>
    <TabsTrigger value="curl">cURL</TabsTrigger>
    <TabsTrigger value="fetch">fetch</TabsTrigger>

  <TabsContent value="curl">
    <CodeBlock lang="bash" allowCopy={true}>
```bash
curl "https://api.phase.sh/public-api/v1/apps/<app_id>/reports/events/overview" \
  -H "Authorization: Bearer phase_private_your_key"
```
    </CodeBlock>
  </TabsContent>
  <TabsContent value="fetch">
    <CodeBlock lang="ts" allowCopy={true}>
```ts
const response = await fetch(
  'https://api.phase.sh/public-api/v1/apps/<app_id>/reports/events/overview',
  {
    headers: {
      Authorization: 'Bearer phase_private_your_key',
    },
  }
);

const data = await response.json();
console.log(data);
```
    </CodeBlock>
  </TabsContent>
</Tabs>

## Request reference

### Path parameters

<TypeTable
  type={{
    appId: {
      description: 'The app ID shown in the dashboard API Keys screen.',
      type: 'string',
      required: true,
    },
  }}
/>

### Headers

<TypeTable
  type={{
    Authorization: {
      description: 'Bearer API key header. Example: `Bearer phase_private_your_key`.',
      type: 'string',
      required: true,
    },
  }}
/>

### Query parameters

<TypeTable
  type={{
    startDate: {
      description: 'Inclusive start date in ISO 8601 format for timeseries and date-filtered breakdowns.',
      type: 'string',
      required: false,
    },
    endDate: {
      description: 'Inclusive end date in ISO 8601 format for timeseries and date-filtered breakdowns.',
      type: 'string',
      required: false,
    },
    metric: {
      description: 'Used by timeseries endpoints. Supported values depend on the endpoint.',
      type: 'string',
      required: false,
    },
    dimension: {
      description: 'Used by breakdown endpoints. Supported values depend on the endpoint.',
      type: 'string',
      required: false,
    },
    limit: {
      description: 'Maximum number of rows returned by a breakdown endpoint.',
      type: 'number',
      required: false,
      default: '10',
    },
  }}
/>

## Report reference

### Events

<TypeTable
  type={{
    'GET /apps/:appId/reports/events/overview': {
      description: 'Returns headline event totals and 24h change values.',
      type: 'overview',
    },
    'GET /apps/:appId/reports/events/timeseries': {
      description: 'Returns daily event counts. Supports `startDate` and `endDate`.',
      type: 'timeseries',
    },
    'GET /apps/:appId/reports/events/breakdown': {
      description: 'Returns event counts grouped by `eventName` or `screenName`. Supports `dimension`, `limit`, `startDate`, and `endDate`.',
      type: 'breakdown',
    },
  }}
/>

### Sessions

<TypeTable
  type={{
    'GET /apps/:appId/reports/sessions/overview': {
      description: 'Returns total sessions, active sessions, average session duration, and bounce rate.',
      type: 'overview',
    },
    'GET /apps/:appId/reports/sessions/timeseries': {
      description: 'Returns daily session metrics. Supported `metric` values: `sessionCount`, `avgSessionDuration`, `bounceRate`.',
      type: 'timeseries',
    },
    'GET /apps/:appId/reports/sessions/breakdown': {
      description: 'Returns session counts grouped by `platform`, `country`, or `city`. Supports `dimension`, `limit`, `startDate`, and `endDate`.',
      type: 'breakdown',
    },
  }}
/>

### Users

<TypeTable
  type={{
    'GET /apps/:appId/reports/users/overview': {
      description: 'Returns total users, active users, new users, and platform/location summaries.',

    },

      description: 'Returns daily user metrics. Supported `metric` values: `activeUsers`, `totalUsers`, `newUsers`.',

    'GET /apps/:appId/reports/users/breakdown': {

## Response metadata

Every report response includes a `meta` object.

<TypeTable
  type={{
    generatedAt: {
      description: 'The server-side timestamp when the response was generated.',
      type: 'string',
    },
    consistency: {
      description: 'The reporting consistency model. Current value: `eventual`.',
      type: 'string',
    },
    identityModel: {
      description: 'The public reporting identity label. Current value: `user`.',
      type: 'string',
    },
  }}
/>

## Key lifetime

When you create a key in the dashboard, choose one of these expiration options:

- 1 week
- 1 month
- 1 year
- Never

## Security guidance

<Callout type="warn">
Do not use API keys in mobile apps, browser bundles, or any other untrusted client. Use them from your own backend, jobs, or internal tooling.
</Callout>
