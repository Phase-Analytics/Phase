## Public API tokens

Public API requests use **app-scoped read-only bearer tokens**.

These tokens are separate from your SDK API key:

- SDK API key → sends analytics data to Phase
- Public API token → reads analytics data from Phase

## Creating a token

1. Open your app in the Phase dashboard
2. Go to **Application → API Keys**
3. Create a Public API token
4. Copy the token immediately

The token is only shown once.

## Authorization header

Send the token as a bearer token:

```bash
curl -X GET "https://api.phase.sh/public-api/v1/apps/<app_id>/capabilities" \
  -H "Authorization: Bearer phase_pat_your_token"
```

## Scopes

Public API tokens can be scoped.

The current external MVP uses:

- `reports:read`

Additional read scopes exist in the broader product plan, but the current public slice is intentionally focused on curated reports and capabilities.

A request fails with `403` if the token does not include the required scope.

## Expiry and revocation

Tokens may have an optional expiration time and can be revoked from the dashboard.

After revocation, any clients using the token will fail on subsequent requests.

## Security notes

- Store tokens securely
- Do not embed Public API tokens in untrusted client applications
- Rotate or revoke tokens if you suspect misuse
- The dashboard stores only a hash of the token at rest
