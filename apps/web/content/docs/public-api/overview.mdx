Phase Public Analytics API gives you read-only, app-scoped access to curated analytics data for external dashboards, scripts, and integrations.

## Overview

Public API v1 is built around:

- app-scoped bearer tokens
- curated report endpoints
- honest device-based semantics
- a deliberately small external surface

## Identity model

Phase Public API v1 is **device-based**.

When you read device metrics, they remain device metrics. They are not renamed to users.

## Report families

Current v1 report families:

- Events
  - overview
  - timeseries
  - breakdown
- Sessions
  - overview
  - timeseries
- Devices
  - overview
  - timeseries
  - breakdown

## Example endpoints

```bash
GET /public-api/v1/apps/:appId/reports/events/overview
GET /public-api/v1/apps/:appId/reports/events/timeseries
GET /public-api/v1/apps/:appId/reports/events/breakdown
GET /public-api/v1/apps/:appId/reports/sessions/overview
GET /public-api/v1/apps/:appId/reports/sessions/timeseries
GET /public-api/v1/apps/:appId/reports/devices/overview
GET /public-api/v1/apps/:appId/reports/devices/timeseries
GET /public-api/v1/apps/:appId/reports/devices/breakdown
```

## Consistency

Phase uses an eventually consistent analytics pipeline. Responses include metadata so clients can understand that the reporting surface is not a strict realtime warehouse.
