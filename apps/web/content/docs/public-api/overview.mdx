Use the Public Analytics API to read Phase report data from your own scripts, dashboards, and backend services.

<Callout type="info">
Public API tokens are app-scoped and read-only. Create them in <strong>Application → API Keys</strong> in the Phase dashboard.
</Callout>

## Base URL

All requests use the same base URL:

<CodeBlock lang="txt" allowCopy={true}>
```txt
https://api.phase.sh/public-api/v1
```
</CodeBlock>

## What you can query

The current API surface includes these report endpoints:

- **Events**
  - overview
  - timeseries
  - breakdown
- **Sessions**
  - overview
  - timeseries
  - breakdown
- **Users**
  - overview
  - timeseries
  - breakdown

## Make your first request

Use a Public API token that starts with `phase_public_`.

<Tabs defaultValue="curl" groupId="public-api-first-request" persist>
  <TabsList>
    <TabsTrigger value="curl">cURL</TabsTrigger>
    <TabsTrigger value="fetch">fetch</TabsTrigger>
  </TabsList>
  <TabsContent value="curl">
    <CodeBlock lang="bash" allowCopy={true}>
```bash
curl "https://api.phase.sh/public-api/v1/apps/<app_id>/reports/events/overview" \
  -H "Authorization: Bearer phase_public_your_token"
```
    </CodeBlock>
  </TabsContent>
  <TabsContent value="fetch">
    <CodeBlock lang="ts" allowCopy={true}>
```ts
const response = await fetch(
  'https://api.phase.sh/public-api/v1/apps/<app_id>/reports/events/overview',
  {
    headers: {
      Authorization: 'Bearer phase_public_your_token',
    },
  }
);

const data = await response.json();
console.log(data);
```
    </CodeBlock>
  </TabsContent>
</Tabs>

```json
{

  "events24hChange": 7.1,
  "meta": {

    "consistency": "eventual",
    "identityModel": "user"
  }
}
```
</CodeBlock>

## Response metadata

Every report response includes a `meta` object.

- `generatedAt` tells you when the response was produced.
- `consistency` is `eventual`.
- `identityModel` is `user`.

## Available endpoints

### Events

- `GET /apps/:appId/reports/events/overview`
- `GET /apps/:appId/reports/events/timeseries`
- `GET /apps/:appId/reports/events/breakdown`

### Sessions

- `GET /apps/:appId/reports/sessions/overview`
- `GET /apps/:appId/reports/sessions/timeseries`
- `GET /apps/:appId/reports/sessions/breakdown`

### Users

- `GET /apps/:appId/reports/users/overview`
- `GET /apps/:appId/reports/users/timeseries`
- `GET /apps/:appId/reports/users/breakdown`
