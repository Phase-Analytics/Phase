## Error response format

Phase returns structured error responses.

<CodeBlock lang="json" allowCopy={true}>
```json
{
  "code": "VALIDATION_ERROR",
  "detail": "Date range cannot exceed 90 days"
}
```
</CodeBlock>

## Status codes

- `400 Bad Request` for invalid parameters or unsupported query shapes.
- `401 Unauthorized` for missing, malformed, expired, revoked, or invalid tokens.
- `403 Forbidden` for tokens that cannot access the requested app or resource.
- `429 Too Many Requests` when the rate limit is exceeded.
- `500 Internal Server Error` for unexpected server errors.

## Request limits

The current API limits are:

- Maximum report range: **90 days**
- Maximum breakdown size: **50 rows**
- Default breakdown size: **10 rows**
- Maximum raw page size: **50**

<Callout type="info">
Build your client with these limits in mind. If a request exceeds a guardrail, handle the error and retry with a smaller date range or result size.
</Callout>

## Consistency

Phase analytics responses are eventually consistent.

- Recently ingested data may take a short time to appear in reports.
- Every report response includes a `meta` object.
- `meta.generatedAt` tells you when the response was created.
- `meta.identityModel` is always `device`.
