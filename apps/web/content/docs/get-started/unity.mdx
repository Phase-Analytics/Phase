## Setup

Initialize once, then identify before tracking. With `AutoBootstrap` enabled (default), the SDK creates a `DontDestroyOnLoad` lifecycle hook for pause/resume and flush-on-quit.

<CodeBlock
  title="GameBootstrap.cs"
  lang="csharp"
  icon={<img src="https://cdn.simpleicons.org/unity/FFFFFF" alt="Unity" className="size-4 dark:invert" />}
>
```csharp
using Phase.Analytics;
using Phase.Analytics.Config;
using Phase.Analytics.Models;
using UnityEngine;

public class GameBootstrap : MonoBehaviour
{
    private async void Start()
    {
        var ok = await PhaseAnalytics.InitializeAsync(new PhaseConfig
        {
            ApiKey = "phase_xxx",
        });

        if (!ok) return;

        await PhaseAnalytics.IdentifyAsync();
        PhaseAnalytics.Track("app_opened");
    }
}
```
</CodeBlock>

You can attach the sample script from `Samples~/PhaseAnalyticsSample` in the package, or copy the pattern above.

## Usage

### Identify

**Required** before `Track`. Registers the device and starts a session.

<CodeBlock lang="csharp" icon={<img src="https://cdn.simpleicons.org/unity/FFFFFF" alt="Unity" className="size-4 dark:invert" />}>
```csharp
await PhaseAnalytics.IdentifyAsync();

await PhaseAnalytics.IdentifyAsync(new DeviceProperties
{
    ["user_id"] = "123",
    ["plan"] = "premium",
});
```
</CodeBlock>

**Privacy by default:** no PII unless you pass properties. If you add PII, obtain consent first.

Properties must be flat primitives: `string`, `number`, `bool`, or `null`.

### Track Events

<CodeBlock lang="csharp" icon={<img src="https://cdn.simpleicons.org/unity/FFFFFF" alt="Unity" className="size-4 dark:invert" />}>
```csharp
PhaseAnalytics.Track("level_complete", new EventParams
{
    ["level"] = 5,
    ["score"] = 1200,
});
```
</CodeBlock>

**Event naming:** alphanumeric, `_`, `-`, `.`, `/`, space; 1–256 characters.

**Event parameters:** same rules as React Native / Swift SDKs (max 32 keys, 8 KB serialized, primitives only).

### Scene and level tracking

There is **no** automatic navigation or screen tracking (unlike Expo/React Native). Fire events manually:

<CodeBlock lang="csharp" icon={<img src="https://cdn.simpleicons.org/unity/FFFFFF" alt="Unity" className="size-4 dark:invert" />}>
```csharp
PhaseAnalytics.Track("scene_loaded", new EventParams { ["scene"] = sceneName });
```
</CodeBlock>

### Clear local data

<CodeBlock lang="csharp" icon={<img src="https://cdn.simpleicons.org/unity/FFFFFF" alt="Unity" className="size-4 dark:invert" />}>
```csharp
await PhaseAnalytics.ClearLocalDataAsync();
// Call InitializeAsync + IdentifyAsync again before tracking
```
</CodeBlock>

## How It Works

### Offline Support

Events are stored under `Application.persistentDataPath` when offline or when requests fail. The queue syncs after `IdentifyAsync` when the network is available.

### Privacy

- No personal data by default
- Device IDs generated locally (ULID)
- `platform` is not sent from Unity (server uses unknown/null)
- Geolocation from IP when `UserLocale` is enabled

### Performance

- `Track` is non-blocking
- Rate limit: 15 events per second
- 50 ms deduplication for identical name + params
- Batch flush with gzip for 100+ queued items

## IL2CPP and release builds

Before shipping production iOS/Android builds:

- Use **IL2CPP** scripting backend
- Keep package `link.xml` (preserves `Phase.Analytics` + Newtonsoft.Json)
- Run a release device smoke test: Initialize → Identify → Track

## Troubleshooting

| Issue | What to check |
|-------|----------------|
| `has no meta file, but it's in an immutable folder` | Pin a commit that includes committed `.meta` files under `packages/phase-unity`; re-import the package |
| `CS0246` / `Phase` namespace not found | `Phase.Analytics` assembly did not compile (often missing `.meta` on git UPM); fix import warnings first |
| `CS8773` file-scoped namespace | Upgrade to `#v0.1.1`+ (asmdef `langVersion: 10`); do not rely on consumer `csc.rsp` |
| Initialize returns false | API key prefix `phase_`, HTTPS `BaseUrl` |
| No events | `IdentifyAsync` before `Track` |
| 401 | Invalid API key |
| Queue not flushing | Online + identified |
| IL2CPP issues | `link.xml`, Newtonsoft package, device release build |
